Notes about the Key Word This

This can refer to many different things in Javascript depending on how it is used.

this within any method refers to the object itself.

When you create a new instance of an object with the new keyword, this refers to the instance.

When this is called outside of any function, in a global context, this defaults to the Window object in the browser.

Objects Are Copied By Reference

An important concept to get your head around is that objects are assigned by reference. This means that if a variable is assigned to an object that already exists, it will simply point to the exact same space in memory. So any changes made using either reference will affect the same object. JSON is a string representation of the object literal notation that we have just seen. There are, however, a couple of key differences:

  1. Property names must be double-quoted
  2. Permitted values are double-quoted strings, numbers, true, false, null, arrays and objects
  3. Functions are not permitted values

Getter Methods

The properties of date objects are unable to be viewed or changed directly. Instead, they have a number of methods known as getter methods, which return information about the date object, such as the month and year. Once you’ve created a date object it will have access to all the getter methods. There are two versions of most methods – one that returns the information in local time, and the other that uses Coordinated Universal Time (UTC). The getTime(), getTimezoneOffset() and getYear() methods don't have UTC

Be Careful Updating className

Changing the className property of an element by assignment will overwrite all other classes that have already been set on the element.

This problem can be avoided by using the classList property instead.

Creating An Element

The document object has a createElement() method that takes a tag name as a parameter and returns that element.

For example, we could create a new list item as a DOM fragment in memory by writing the following in the console:

Chapter 7 Summary

  1. Events occur when a user interacts with a web page.
  2. An event listener is attached to an element, then invokes a callback function when the event occurs.
  3. The event object is passed to the callback function as an argument, and contains lots of properties and methods relating to the event.
  4. There are many types of event, including mouse events, keyboard events, and touch events.
  5. You can remove an event using the removeEventListener method.
  6. The default behavior of elements can be prevented using the preventDefault() function.
  7. Event propagation is the order the events fire on each element.
  8. Event delegation is when an event listener is added to a parent element to capture events that happen to its children elements.
